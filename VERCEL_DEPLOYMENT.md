# 🚀 VERCEL + RAILWAY DEPLOYMENT GUIDE (HTTPS)

## Overview
- **Frontend**: Deploy on Vercel (React + Vite) ✅ HTTPS built-in
- **Backend**: Deploy on Railway (Spring Boot + Java) ✅ HTTPS built-in
- **Database**: Deploy on Railway (MySQL 8.0) ✅ Managed

**Total Cost**: FREE tier available for both platforms!

---

## 📋 Architecture

```
User Browser
    ↓
    ├─→ https://your-app.vercel.app (Frontend - Vercel)
    │   └─→ API calls to Backend
    │
    └─→ https://backend-xxxx.railway.app (Backend - Railway)
        └─→ Database (Railway MySQL)
```

## ⚡ STEP 1: Prepare Frontend for Vercel

### 1.1 Update Frontend Configuration

Create `ims-frontend/vercel.json`:
```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "env": {
    "VITE_API_URL": "@VITE_API_URL"
  }
}
```

### 1.2 Update API URL in Frontend

Create `ims-frontend/.env.vercel`:
```
VITE_API_URL=https://your-backend-url.railway.app/api
```

Update `ims-frontend/src/services/api.js`:
```javascript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:9001/api';

export const api = axios.create({
  baseURL: API_BASE_URL,
  headers: {
    'Content-Type': 'application/json'
  }
});

// Add token to requests
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
```

### 1.3 Ensure Build Works Locally
```bash
cd ims-frontend
npm run build
npm run preview
```

## 🚀 STEP 2: Deploy Frontend on Vercel

### 2.1 Create Vercel Account
1. Go to **https://vercel.com**
2. Sign up with GitHub, GitLab, or Bitbucket
3. Link your repository

### 2.2 Push Code to GitHub

```bash
# From project root
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/nexus-ims-pro.git
git branch -M main
git push -u origin main
```

### 2.3 Deploy on Vercel

**Option A: Via GitHub (Recommended)**
1. Go to https://vercel.com/new
2. Select "Import Git Repository"
3. Choose your GitHub repo
4. Select "ims-frontend" as root directory
5. Click "Deploy"

**Option B: Via CLI**
```bash
npm install -g vercel
cd ims-frontend
vercel --prod
```

### 2.4 Configure Environment Variables in Vercel

1. Go to **Vercel Dashboard** → Your Project → **Settings** → **Environment Variables**
2. Add:
```
VITE_API_URL = https://your-backend-railway-url.railway.app/api
```

3. Redeploy to apply changes:
```bash
vercel --prod
```

**Result**: Your frontend is now live at `https://your-app.vercel.app` ✅

## 🚀 STEP 3: Deploy Backend on Railway

### 3.1 Create Railway Account
1. Go to **https://railway.app**
2. Sign up with GitHub
3. Connect your GitHub account

### 3.2 Setup Railway Project

**Option A: Import from GitHub (Recommended)**
1. Go to https://railway.app/dashboard
2. Click **"+ New"** → **"GitHub Repo"**
3. Select your repo
4. Railway auto-detects Java/Spring Boot
5. Configure MySQL database (see next step)

**Option B: Deploy Using CLI**
```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# Initialize
cd ims-backend
railway init

# Configure
railway link <project-id>

# Deploy
railway up
```

### 3.3 Add MySQL Database to Railway

1. In Railway Dashboard → Your Project
2. Click **"+ New"**
3. Select **"Database"** → **"MySQL"**
4. Railway creates MySQL automatically
5. Copy connection details

### 3.4 Set Environment Variables in Railway

1. Go to **Variables** tab in Railway
2. Add the following:

```
DB_URL=mysql://root:PASSWORD@HOST:PORT/railway
DB_USER=root
DB_PASSWORD=<auto-generated>
JWT_SECRET=your-production-jwt-secret-key-minimum-64-characters-recommended
```

**Get MySQL details from Railway Dashboard:**
```
Hostname: ${{MYSQL_HOST}}
Port: ${{MYSQL_PORT}}
Username: ${{MYSQL_USER}}
Password: ${{MYSQL_PASSWORD}}
Database: ${{MYSQL_DB}}
```

### 3.5 Update Backend Configuration

Create `ims-backend/src/main/resources/application-railway.properties`:
```properties
spring.application.name=ims

# Database Configuration - Railway MySQL
spring.datasource.url=jdbc:mysql://${DB_HOST}:${DB_PORT}/${DB_NAME}?createDatabaseIfNotExist=true&useSSL=true&serverTimezone=UTC
spring.datasource.username=${DB_USER}
spring.datasource.password=${DB_PASSWORD}
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# Hibernate Configuration
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=false
spring.jpa.properties.hibernate.format_sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQLDialect

# JWT Configuration
ims.jwt.secret=${JWT_SECRET}
ims.jwt.expiration=86400000

# Server Configuration
server.port=${SERVER_PORT:8080}

# CORS Configuration
spring.web.cors.allowed-origins=https://your-app.vercel.app,https://your-domain.com
spring.web.cors.allowed-methods=GET,POST,PUT,DELETE,OPTIONS
spring.web.cors.allowed-headers=*
spring.web.cors.allow-credentials=true
```

### 3.6 Create Procfile for Railway

Create `Procfile` in project root:
```
web: cd ims-backend && mvn spring-boot:run -Dspring-boot.run.arguments="--spring.profiles.active=railway"
```

Or create `ims-backend/Procfile`:
```
web: java -jar target/ims-0.0.1-SNAPSHOT.jar --spring.profiles.active=railway
```

### 3.7 Deploy Backend

**Method 1: Automatic (GitHub)**
- Push changes to GitHub
- Railway auto-deploys

**Method 2: Manual**
```bash
railway up
```

**Wait for deployment to complete...**

After successful deployment, Railway provides:
- Backend URL: `https://ims-backend-xxxx.railway.app`
- MySQL Connection string

**Result**: Your backend is now live at `https://ims-backend-xxxx.railway.app` ✅

## 🚨 TROUBLESHOOTING

### Frontend Deployment Issues

**Issue: Build fails on Vercel**
```
Solution:
1. Check build locally: npm run build
2. Verify Node version: 18+ required
3. Check all dependencies installed
4. View Vercel logs for details
```

**Issue: API calls return 404**
```
Solution:
1. Check VITE_API_URL environment variable
2. Verify backend is running
3. Check CORS configuration
4. Redeploy frontend
```

**Issue: CORS errors**
```
Solution:
1. Update SPRING_WEB_CORS_ALLOWED_ORIGINS in Railway
2. Ensure full URL including https://
3. Wait for Railway auto-restart
4. Clear browser cache
```

### Backend Deployment Issues

**Issue: Build fails on Railway**
```
Solution:
1. Check Java version: JDK 17 required
2. Verify Maven build works locally
3. Check pom.xml dependencies
4. View Railway deployment logs
```

**Issue: Database connection failed**
```
Solution:
1. Get MySQL credentials from Railway Dashboard
2. Update environment variables
3. Check database is running
4. Verify connection string format
```

**Issue: PORT binding error**
```
Solution:
1. Ensure server.port is set to ${SERVER_PORT:8080}
2. Don't hardcode port to 9001
3. Let Railway assign the port
```

### Common Issues

**Blank page after login**
```
1. Check API_URL is correct
2. Verify JWT token is stored
3. Check localStorage in DevTools
4. View browser console for errors
```

**Slow response times**
```
1. Check Railway resource allocation
2. Add database indexes
3. Optimize API queries
4. Enable caching
```
