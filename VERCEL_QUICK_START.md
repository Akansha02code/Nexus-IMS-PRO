### STEP 2: Deploy Backend on Railway (5 minutes)

#### 2a. Create Railway Account
1. Go to https://railway.app
2. Click **"Start New Project"**
3. Sign in with GitHub

#### 2b. Create New Project
1. Click **"+ New Project"**
2. Select **"GitHub Repo"**
3. Choose your `nexus-ims-pro` repository
4. Wait for Railway to scan (it auto-detects Java)

#### 2c. Add MySQL Database
1. In Railway Dashboard, click **"+ New"**
2. Select **"Database"** → **"MySQL"**
3. Wait for database to initialize (~2 minutes)

#### 2d. Set Environment Variables
1. Click on **"ims-backend"** service
2. Go to **"Variables"** tab
3. Add these variables:

```
DB_HOST=mysql.railway.internal
DB_PORT=3306
DB_NAME=railway
DB_USER=root
DB_PASSWORD=<copy from MySQL service>
JWT_SECRET=your-super-secure-jwt-secret-key-minimum-64-chars-long-generate-random
CORS_ORIGINS=https://your-app.vercel.app
SPRING_PROFILES_ACTIVE=railway
```

**To find MySQL password:**
1. Click on **"MySQL"** service in Railway
2. Copy password from Variables tab

#### 2e. Copy Backend URL
1. Click on **"ims-backend"** service
2. Copy the **"Public URL"** (e.g., `https://nexus-ims-prod-xxxx.railway.app`)
3. **Save this - you'll need it for frontend!**

---

### STEP 3: Deploy Frontend on Vercel (5 minutes)

#### 3a. Create Vercel Account
1. Go to https://vercel.com
2. Click **"Sign Up"**
3. Sign in with GitHub

#### 3b. Create New Project
1. Click **"Add New..."** → **"Project"**
2. Select your GitHub repository
3. Select **Root Directory: `ims-frontend`**
4. Click **"Deploy"**

#### 3c. Set Environment Variables
1. Go to **Settings** → **Environment Variables**
2. Add:
```
VITE_API_URL = https://YOUR-RAILWAY-BACKEND-URL/api
```
(Replace with the URL you copied from Railway)

3. Redeploy: Click **"Deployments"** → Last deploy → **"Redeploy"**

## 🚨 TROUBLESHOOTING

### ❌ Frontend shows blank page or 404

**Solution:**
1. Check `VITE_API_URL` is set correctly in Vercel
2. Make sure URL ends with `/api`
3. Redeploy on Vercel: **Deployments** → Last deploy → **Redeploy**
4. Clear browser cache: Ctrl+Shift+Delete
5. Check browser console (F12) for errors

### ❌ API calls return 404 or CORS error

**Solution:**
1. Backend might still be building (takes 2-3 minutes)
2. Check backend status in Railway: should say "Up"
3. Test manually: `curl https://your-backend-xxxx.railway.app/api/health`
4. Verify `CORS_ORIGINS` in Railway includes `https://your-app.vercel.app`
5. Wait 2 minutes for Railway to apply changes

### ❌ Login returns error

**Solution:**
1. Check if database was created: Railway Dashboard → MySQL → check status
2. Backend should have auto-created tables
3. Check Railway logs: Click backend service → **Logs** tab
4. Try refreshing the page
5. Clear browser cache and localStorage

### ❌ Database connection failed

**Solution:**
1. In Railway, click MySQL service
2. Verify it says "Up" (green status)
3. Copy exact password and paste in `DB_PASSWORD`
4. Make sure no extra spaces in variable values
5. Click "Restart" on MySQL service

### ❌ Backend build fails on Railway

**Solution:**
1. Go to Railway Dashboard → ims-backend service → **Logs**
2. Look for build error
3. Common issues:
   - Java version wrong (need JDK 17+)
   - Maven build fails
   - Missing dependencies
4. Click "Rebuild": Railway → service → **"Rebuild"**

### Viewing Logs

**Vercel Logs:**
1. Vercel Dashboard → Deployments
2. Click on a deployment
3. View logs

**Railway Logs:**
1. Railway Dashboard → select service
2. Click "Logs" tab
3. Real-time logs shown

### Restart Services

**Railway (if stuck):**
1. Railway Dashboard → select service
2. Click "Restart"
3. Wait 1-2 minutes for restart

**Vercel:**
1. Deployments → click deploy
2. Click "..." → "Redeploy"

**Vercel:**
1. Settings → Domains
